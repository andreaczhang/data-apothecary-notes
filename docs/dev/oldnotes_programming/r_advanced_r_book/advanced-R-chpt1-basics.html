Names and values

x <- c(1, 2, 3)
  • the code creates an object c(1, 2, 3)
  • it binds the object to a name x
y <- x

a name is a reference to a value (object c(1, 2, 3) does not need to have a name). y <- x get a binding to the same existing object.

Vectors

2 types: atomic vector + list

atomic vector

  • logical
  • integer
  • double
  • character (include string)

they are of such type: c(1, 2, 3), and can be tested by is.atomic(), is.recursive(). Recursive is a list.

list

List is different from an atomic vector in

  • lists are recursive (nested)

  • lists can hold various types of vectors, and vector can only have one type

attributes and S3 objects

Matrix, array, factor, date-time are built on top of atomic vectors by adding attributes

can give as many as we want, by

a <- 1:3
attr(a, 'meta1') <- 'feature1'
# check by 
attributes(a) # will print out a list

class attribute and S3 object

4 important types of S3 vectors

  • categorical data (factor)
  • dates (day resolution), Date vector
  • date-times (second resolution), stored in POSIXct vector
  • duration, difftime vector
df <- data.frame(x = 1:3, y = letters[1:3])

# difference between the following
typeof(df)  # list
attributes(df) # names: 'x','y', class: data.frame, row.names: 1,2,3
class(df) # data.frame
  • typeof() checks the vector type: if it is atomic vector, it should return integer, character, double, logical. If it is a dataframe or list, it returns list
  • attributes() returns all the attributes (everything that is not the data itself: rownames, names, etc)
  • class() is one of the attributes for S3 object, which can be datetime, factor, data.frame etc.
    • note that for a numeric vector it might behave differently: returns numeric, while when checking attributes there is none. So class can work with NULL attributes.

Subsetting

It is recommended to use [[]] also with vectors, because this guarantees getting one element (in list the behavior of [] is less obvious)

Can always use purrr::pluck() to subset list.

Some examples:

# 1. lookup tables (char subset)
x <- c('m', 'f', 'u', 'm', 'm', 'f')
lookup <- c(m = 'Male', 
            f = 'Female',
            u = NA)
lookup['m']
# look up the whole x vec
lookup[x]
# this returns the whole Male, Female vector. do not need CASEWHEN!
lookup[x] %>% unname # this gives only result, not m,f colnames

Random permutation in columns and rows are also useful.

# 2. bootstrap 
df <- data.frame(x = c(1,2,3,1,2), 
                 y = 5:1, 
                 z = letters[1:5])

# randomly reorder (sample by default is WITHOUT replacement)
df[sample(nrow(df)), ]

# 3 random rows 
df[sample(nrow(df), 3), ]

# 10 bt replicates (if replace = F and replicates greater than nrow, will print error)
df[sample(nrow(df), 10, replace = T), ]


# 2.1 permute columns (technique used in random forest )
df[, sample(1:ncol(df))]

Control flow

Choices (if, switch)

if, ifelse, case_when

fun2 <- function(x){
  if(x == '1'){
    'option 1'
  }else if(x == '2'){
    'option 2'
  }else{
    stop('no option')
  }
}
fun2(2)

# with switch: can avoid many else ifs
fun2switch <- function(x){
  switch(x, 
         '1' = 'option 1',  # recommended to use string, not numeric
         '2' = 'option 2', 
         stop('no option'))  # last one should always be a stop, otherwise nothing prints
}

fun2switch(3)

Loops (for, while)

instead of use for i in 1:length(x), can use for i in seq_along(x), which gives 1:length(x) but avoiding the issue of zero-length x

# x2 has 0 length
x2 <- c()
for (i in 1:length(x2)){
  cat(i, '\n')
}
# 1, 0

for (i in seq_along(x2)){
  cat(i, '\n')
}
# prints nothing, as should be

while

I rarely use while, because I always know how many times to iterate over.

while is more flexible than for

Functions

3 components:

  • arguments, formals()
  • body, body()
  • environment environment(). based on where I define the function

3 types of functions in R (when checking typeof():

  • closure (can have 0 or more arguments found by formals())
  • builtin (is.primitive() returns T)
  • special (is.primitive() returns T)

when checking the arguments of the latter two using formals(), it will return NULL. It does not mean the function doesn’t take any arguments!

f1 <- function(x,y){
  # plus
  x+y
}
typeof(f1)  # closure (function)
class(f1)   # function

typeof(sum) # built in
typeof(`[`) # special 
# these do not have the three components 
is.primitive(sum)
is.primitive(`[`)
is.primitive(mean)  # not primitive!

with args(func), formals(func) can get the minimum arguments for the function

Conditions

3 conditions: errors, warnings, messages

errors

stop() or rlang::abort().

To ignore: try(). The line will print an error, but execution continues

warnings

warning(). To ignore: suppressWarnings()

messages

warning(). To ignore: suppressMessages()

f4e <- function(x){
  # log(x)
  try(log(x))    # not tryCatch
  message('see if the execution carries out')
}
f4e('a')

suppressMessages(f4e('a'))  # in this case the message is suppressed
suppressWarnings(f4e('a'))  # there is no warning, so nothing changes

handlers

tryCatch() for error

fail_with <- function(expr, value = NULL){
  tryCatch(
    error = function(cnd) value,  # this function here is necessary
    expr
  )
}
fail_with(log(10))   # execute normally
fail_with(log('x'))  # here value is prompted
fail_with(log('x'), value = NA)   # returns NA

Difference between try() and tryCatch(): if the former meets an error, it passes onto the next code chunk to execute. The later will modify the behaviour of the exit, for example, give a specific value if an error occurs.

withCallingHandlers() for warning and messages

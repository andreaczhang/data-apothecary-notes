git add mars.txt
git diff  # compares working area with staging area, disappears after git add, before commit
git diff --staged  # comparing staging area with repositories
git diff HEAD # working area vs repositories

if I have a separate file called mars2.txt, and I do git diff, nothing will happen until I do git add.

It’s possible that changes in both files are displayed at the same time, can be seen using git status and git diff, and git add . will update both.

ignore

before committing, can use .gitignore to specify files unneeded. Do it before add anything. Add these lines in the file to ignore the hidden directories.

.*
!/.gitignore

unstaged, use

$ git checkout -- <file>
$ git checkout '*.DS_Store'

to discard changes in working directory.

or

$ git reset HEAD

checkout

git checkout is switching between branches in a repo you have, for example if you’re on master branch and want to go to develop_branch, git checkout develop_branch.

git clone is fetching repo you don’t have

Using another computer

For the project, I have re-set up the github account on the pink computer. Then, go to the parent directory (‘Work’) and do

git clone ### then add the .git file that appears

After any changes made in the pink computer, push as usual.

In this computer, git pull. This will make sure the changes are synced.

When syncing once I’m back from vacation:

(on the main computer) first try git pull

It will say there are some files that are untracked, and modified locally.

Modified locally: force over write (i.e. ignore all local changes) by

git reset --hard
git pull

Untracked:

git clean -i
c # for clean

In the end pull again.

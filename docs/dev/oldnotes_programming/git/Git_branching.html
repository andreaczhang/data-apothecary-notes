Branching and merging

CAUTION: it is important to know which branch you are, and which to merge (from)!

  • git merge master: update branch from master

  • git merge branch: update master from branch

Branches

Type 1 branch: local branches

Type 2 branch: remote-tracking branches

Create and switch to branch

git checkout -b my_new_branch

# what it does:: 
git branch my_new_branch
git checkout my_new_branch

master is ahead of branch:

Update the branch.

git checkout my_new_branch  # go to branch
git status # this branch is still up to date with origin/my_new_branch, even though it is behind master

Can check the difference between these two. Note that the order matter: these two are not the same!

git diff my_new_branch master 
git diff master my_new_branch

merge into master (even though branch is behind master!)

git merge master  # when on branch

At this stage, the branch will be ahead of the remote branch origin/my_new_branch. Now need to push the updates through, to make the remote branch updated too.

branch is ahead of master

Option 1: merge branch into master locally, then push master

Need to check difference to make sure everything is correct.

git merge my_new_branch  # when on master.

Note: from network of branches, this merge will NOT appear as the merge is done locally.

Option 2: push to remote branch, pull request, and merge into master

On branch, NOT master!

  • If push to branch (NOT master, WITHOUT merge) at this point, the changes will appear on github under this branch (remote), which requires a pull request.
  • If merge master (on branch), nothing will happen.

delete branch (after pull requested and merging from remote)

git fetch -p 

# check which branch is there
git branch
git branch --merged # which merged branch we can delete
